## 2.1 FSM 时间窗口增强（Fail Density Control）

### 问题
v1.3 FSM 仅基于“次数”，无法区分：
- 高频错误（危险）
- 低频错误（可接受）

---

### 解决方案

引入时间窗口：

fail_count → fail_count_in_window(T)

---

### 定义

设时间窗口 T（例如 10 秒）：

fail_rate = failures within last T seconds

---

### 触发规则

- 若 fail_rate ≥ N_fail → HARD_FAIL
- 若单次错误 → SOFT_FAIL

---

### 实现建议（伪代码）

```python
from collections import deque
import time

class TimeAwareFSM:
    def __init__(self, n_fail=5, m_recover=3, window_sec=10):
        self.state = "NORMAL"
        self.fail_timestamps = deque()
        self.success_count = 0
        self.N_fail = n_fail
        self.M_recover = m_recover
        self.window = window_sec

    def update(self, has_error):
        now = time.time()

        # 清理窗口外数据
        while self.fail_timestamps and now - self.fail_timestamps[0] > self.window:
            self.fail_timestamps.popleft()

        if has_error:
            self.fail_timestamps.append(now)
            self.success_count = 0

            if self.state == "NORMAL":
                self.state = "SOFT_FAIL"

            if len(self.fail_timestamps) >= self.N_fail:
                self.state = "HARD_FAIL"

        else:
            self.success_count += 1

            if self.state == "SOFT_FAIL" and self.success_count >= self.M_recover:
                self.state = "NORMAL"

        return self.state != "NORMAL"
```

## 2.2 η 参数自适应机制（Distribution Adaptive Scaling）

---

### 问题

固定 η = 0.5 假设 σ 稳定，但实际：

- 数据分布会漂移
- σ 会变化

导致：
- 异常通道失去标定意义

---

### 自适应方案

目标保持：

3σ → tanh ≈ 0.9

---

### 推导

η = artanh(target) / (k · σ_effective)

其中：

- target ≈ 0.9
- k = 3（σ倍数）

---

### 简化实现

η ≈ C / rolling_std(x_norm)

---

### 工程规则

- 默认 η = 0.5（冷启动）
- 当 rolling_std 稳定后启用自适应
- η 必须限制范围：

η ∈ [0.2, 1.0]

## 2.3 FSM 多实例隔离（Instance Isolation）

---

### 问题

v1.3 FSM 为全局：

- 单一错误源可污染整个系统
- 不同实体之间状态串扰

---

### 解决方案

FSM 必须按实例隔离：

---

### 定义

FSM_key = (entity_id, feature_group, source_id)

---

### 示例

```python
fsm_registry = {}

def get_fsm(key):
    if key not in fsm_registry:
        fsm_registry[key] = TimeAwareFSM()
    return fsm_registry[key]
```

---

### 工程要求

- 每个独立数据源必须独立 FSM
- 不允许全局共享 FSM 状态
- FSM 生命周期需绑定业务实体
