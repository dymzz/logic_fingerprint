```mermaid
graph TD
    Start[接收到新请求] --> CheckConsensus{检查全局失败共识}
    
    %% 全局失败共识
    CheckConsensus -->|fail_ratio >= threshold| GlobalFail[触发 trigger_global_fail]
    CheckConsensus -->|正常| CheckState{当前 FSM 状态?}
    
    %% 状态流转
    CheckState -->|CLOSED / NORMAL| Process[正常执行业务逻辑]
    CheckState -->|HARD_FAIL| Reject[拦截请求 / 执行降级 fallback]
    
    %% 探针判定
    CheckState -->|HALF_OPEN| CheckProbeCondition{满足 Probe 放行条件?}
    CheckProbeCondition -->|request_counter % N == 0| AllowProbe[放行试探流量]
    CheckProbeCondition -->|now - last_probe >= interval| AllowProbe
    CheckProbeCondition -->|不满足| Reject
    
    %% 试探结果判定
    AllowProbe --> Execute[执行系统调用计算]
    Execute --> ProbeResult{调用是否成功?}
    
    %% 连续成功逻辑
    ProbeResult -->|成功| SuccessInc[success_count += 1]
    ProbeResult -->|失败| SuccessReset[success_count = 0]
    
    SuccessInc --> CheckK{success_count >= K?}
    CheckK -->|是| Recover[状态流转: CLOSED]
    CheckK -->|否| WaitNext[保持 HALF_OPEN, 等待下次探针]
    
    SuccessReset --> ToHardFail[状态回退: 标记为失败并拦截]
