# 🛡️ 逻辑指纹系统·工程落地执行标准 v1.6（稳定性增强版）

## 一、版本升级说明

在 v1.5+ 基础上补强三个关键稳定性问题：

1. 时间驱动 Probe（解决低流量挂死）
2. 连续成功判定（避免误恢复）
3. 全局失败共识机制（避免集群雪崩）

---

## 二、Probe 机制（双驱动）

### 2.1 请求 + 时间 双驱动

```python
allow_probe = (
    request_counter % N == 0
    or (now - last_probe_time >= probe_interval)
)
```

---

## 三、连续成功判定

```python
if probe_success:
    success_count += 1
else:
    success_count = 0

if success_count >= K:
    state = "CLOSED"
```

建议：
- K = 3~5

---

## 四、全局失败共识机制

```text
fail_ratio = fail_nodes / total_nodes

if fail_ratio >= threshold:
    trigger_global_fail
```

建议：
- threshold = 0.2 ~ 0.3

Redis结构：
SET fail_nodes
