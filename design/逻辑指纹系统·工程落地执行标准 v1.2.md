# 🛡️ 逻辑指纹系统·工程落地执行标准 v1.2

---

## 一、系统总体架构（分层设计）

### 1.1 理论基式（解释层）
κ_base = β1 c̃ + β2 ṽ + β3 r̃ - β4 h̃_t - β5 h̃_d

用途：
- 离线验证
- A/B 实验
- 可解释分析

---

### 1.2 部署修正层（抗噪层）
κ_deploy = κ_base - λ_conf * U_conf(t) - λ_cal * U_cal(t)

---

### 1.3 安全熔断层（保护层）
κ_final =
    0                    if trigger_failsafe = 1
    κ_deploy             otherwise

---

## 二、数据预处理标准

### 2.1 标准化方法

- 物理/业务量：x̃ = x / S_mean → clip [0,3]
- 统计/代理量：x̃ = (x-μ)/σ → clip [-3,3]
- 熵：h̃ = H/H_max → [0,1]

### 2.2 强制规则

- 分母为0 → ERR_NORM
- 残差变量按统计量处理
- 所有变量必须标准化+裁剪

---

## 三、不确定性处理

### 3.1 基础项
h̃_t, h̃_d

### 3.2 防脱敏项
U_conf(t) = 1 - |2t - 1|^γ, 0.5 ≤ γ ≤ 0.8

增加变化率限制：
ΔU_conf ≤ ρ

---

### 3.3 校准项
U_cal(t) = |PredictedConfidence - HistoricalAccuracy|

降级：
λ_cal = 0 if calibration unavailable

---

## 四、共线性处理

r' = r - αc

使用 r' 替代 r

---

## 五、时序处理（时间感知 EMA）

α = 1 - exp(-Δt / τ)

t̄ = α t_now + (1 - α) t_prev

双通道：
- Stable：t̄
- Fast：t_now

---

## 六、异常保真机制

主通道：

## 七、安全熔断机制

触发条件：

- ERR_NULL
- ERR_OVERFLOW
- ERR_NORM
- ERR_DIM
- ERR_LOGIC
- ERR_TIMEOUT_CORE
- ERR_TIMEOUT_CAL（降级）
- ERR_SPIKE

---

## 八、熔断恢复协议

状态：
- NORMAL
- SOFT_FAIL
- HARD_FAIL

规则：
- 单次错误 → SOFT_FAIL
- 连续 N 次 → HARD_FAIL
- 成功恢复 → NORMAL

## 九、贡献占比监控

CS_i = |β_i x_i| / Σ|β_j x_j|

触发：
- 单项贡献 > 0.7
- 或偏移 > 2σ
