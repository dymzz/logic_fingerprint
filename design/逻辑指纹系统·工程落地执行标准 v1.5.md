## 二、系统状态机升级（四态 FSM）

状态定义：

- NORMAL：正常运行
- SOFT_FAIL：短期失败保护
- HALF_OPEN：灰度恢复试探
- HARD_FAIL：锁定熔断

---

## 三、HALF_OPEN 灰度试探机制（核心新增）

### 3.1 设计目标

防止系统从 SOFT_FAIL 恢复时出现：

- 雪崩重放（Thundering Herd）
- 下游再次崩溃

---

### 3.2 状态流转

NORMAL → SOFT_FAIL → HALF_OPEN → NORMAL  
                          ↓  
                       HARD_FAIL  

---

### 3.3 策略 B：试探流量（推荐）

在 HALF_OPEN 状态下：

仅放行一部分流量进行恢复验证

---

### 3.4 核心规则

设：

- probe_rate = 1%（默认）
- success_threshold = K
- failure_threshold = F

---

#### 行为：

- 仅允许 1% 请求进入真实下游
- 其余请求继续走降级路径（κ_final = 0 或 fallback）

---

#### 状态判断：

- 若试探流量全部成功（或达到成功阈值）：
  → 进入 NORMAL

- 若试探流量中出现失败：
  → 回退 SOFT_FAIL 或进入 HARD_FAIL

---

### 3.5 工程实现（示意）

```python
import random

def should_probe(probe_rate=0.01):
    return random.random() < probe_rate
```

```python
if fsm.state == "HALF_OPEN":
    if should_probe():
        # 允许请求通过
        pass
    else:
        # 拦截或降级
        return fallback_response
```

---

### 3.6 参数建议

| 参数 | 默认值 | 说明 |
|------|--------|------|
| probe_rate | 0.01 | 试探流量比例 |
| success_threshold | 5 | 连续成功次数 |
| failure_threshold | 1 | 单次失败即回退 |

## 五、η 自适应安全护栏

---

### 5.1 基础公式

η ≈ C / rolling_std(x_norm)

---

### 5.2 安全限制

- std_min = 0.01
- η_default = 0.5
- η ∈ [0.2, 1.0]

---

### 5.3 计算流程

1. 若 std < std_min → η = η_default  
2. 否则 η = C / std  
3. clip η 到范围  
4. EMA 平滑 η
