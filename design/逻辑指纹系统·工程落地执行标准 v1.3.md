# 🛡️ 逻辑指纹系统·工程落地执行标准 v1.3

---

## 一、系统总体架构（分层设计）

### 1.1 理论基式（解释层）
κ_base = β1 c̃ + β2 ṽ + β3 r̃ - β4 h̃_t - β5 h̃_d

---

### 1.2 部署修正层（抗噪层）
κ_deploy = κ_base - λ_conf * U_conf(t) - λ_cal * U_cal(t)

---

### 1.3 安全熔断层（保护层）
κ_final =
    0                    if trigger_failsafe = 1
    κ_deploy             otherwise

---

## 二、数据预处理标准

- 物理/业务量：x̃ = x / S_mean → clip [0,3]
- 统计/代理量：x̃ = (x-μ)/σ → clip [-3,3]
- 熵：h̃ = H/H_max → [0,1]

### 新增：基线漂移监控
drift_ratio = EMA(x) / S_mean

触发：
|drift_ratio - 1| > 0.3 → WARN_BASELINE_DRIFT

---

## 三、不确定性处理

U_conf(t) = 1 - |2t - 1|^γ, 0.5 ≤ γ ≤ 0.8

增加变化率限制：
ΔU_conf ≤ ρ

U_cal(t) = |PredictedConfidence - HistoricalAccuracy|

降级：
λ_cal = 0 if unavailable

---

## 四、共线性处理

r' = r - αc

---

## 五、时序处理（时间感知 EMA）

α = 1 - exp(-Δt / τ)

冷启动保护：
if is_first_frame:
    t_stable = t_now
else:
    t_stable = α t_now + (1 - α) t_prev

双通道：
- Stable：t_stable
- Fast：t_now

---

## 六、异常保真机制

主通道：clip(x)

异常通道：
x_anom = tanh(η x_norm)

参数：
η ≈ 0.5（确保3σ ≈ 0.9输出）

⚠️ 禁止双重计数

---

## 七、安全熔断机制

新增：
ERR_TIME_ROLLBACK（Δt < 0）

完整列表：
- ERR_NULL
- ERR_OVERFLOW
- ERR_NORM
- ERR_DIM
- ERR_LOGIC
- ERR_TIMEOUT_CORE
- ERR_TIMEOUT_CAL（降级）
- ERR_SPIKE
- ERR_TIME_ROLLBACK

---

## 八、熔断恢复协议（防震荡）

状态：
- NORMAL
- SOFT_FAIL
- HARD_FAIL

新增参数：
- N_fail（进入 HARD_FAIL）
- M_recover（恢复阈值）

规则：
- 单次失败 → SOFT_FAIL
- 连续失败 N 次 → HARD_FAIL
- 连续成功 M 次 → NORMAL

---

## 九、贡献占比监控

CS_i = |β_i x_i| / Σ|β_j x_j|

触发：
- 单项 > 0.7
- 或偏移 > 2σ

---

## 十、执行顺序（强制）

1. 输入校验（含时间校验）
2. 标准化
3. 裁剪
4. 残差化
5. EMA（含冷启动）
6. κ_base
7. U_conf（含限速）
8. U_cal（或降级）
9. κ_deploy
10. 熔断判断
11. κ_final

⚠️ 任一前置失败 → 直接熔断

## 十三、核心设计原则

- 分层：解释层 ≠ 部署层 ≠ 安全层
- β 不承担量纲
- 不确定性必须多源建模
- 系统必须可恢复、可降级、可审计
- 防止时间、分布、异常三类污染
