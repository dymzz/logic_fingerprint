# 用户模式示例

这页只收用户模式示例，也就是“以 `@protect()` 为主入口”的示例。

如果你只是想保护函数边界，而不想自己装 runtime、registrar、HTTP 服务，就看这页。

这也是当前默认推荐路线。

如果你想先看最短接入路径，先看：

- `documents/Tutorial/用户模式快速接入.md`

## 什么时候看这页

适合这些场景：

- LangChain `invoke()` / `ainvoke()`
- SDK 调用
- 工具函数调用
- 本地脚本任务

不适合这些场景：

- 平台统一注册多个 handlers
- OpenClaw / Gateway / HTTP 服务接入
- 需要 `handler_registrars` 的服务化启动

## 当前示例

### LangChain

看这里：

- `examples/langchain/user_mode.py`
- `examples/langchain/README.md`

这套示例展示的是：

## 配置怎么放

用户项目里统一放：

```text
your_project/config/config.yaml
```

最小模板可以从这里复制：

- `examples/langchain/config.yaml.example`
- `config/config.yaml.example`

用户模式一般只需要关心：

- `instance_id`
- `default_source`
- `backend_type`
- probe 相关参数

## 入口怎么用

用户模式推荐只用这两个入口：

- `from logicfp import protect`
- `from logicfp import create_protector`

如果你要先排查“为什么配置没生效”，优先看：

- `examples/user_mode/config_diagnostics.py`
- `logicfp.config.describe_effective_config(...)`

如果你发现自己已经开始关心 registrar、runtime、`create_http_app()`，说明你已经在往工程模式走了。
