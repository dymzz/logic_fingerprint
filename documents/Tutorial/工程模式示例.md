# 工程模式示例

这页只收工程模式示例，也就是“以 runtime / registrar / CLI / HTTP 服务为主入口”的示例。

如果你是在保护服务边界，而不是单个函数边界，就看这页。

## 什么时候看这页

适合这些场景：

- HTTP 服务启动
- 多个 handler 统一注册
- OpenClaw / Gateway / ACP 集成
- 需要 `handler_registrars`
- 需要 `memory / redis / redis_ttl` 切换

## 当前示例

### 1. 纯 registrar 模板

- `examples/production_handlers.py`
- `examples/production.config.yaml.example`

适合先理解最小工程接入面：注册两个 handler，直接让 runtime 加载。

### 2. 带依赖装配的 registrar 模板

- `examples/production_services.py`
- `examples/services.config.yaml.example`

适合你已经有 repository、gateway、service 这些依赖对象，想把它们在 registrar 里装起来。

### 3. OpenClaw 接入

- `examples/openclaw/register_handlers.py`
- `examples/openclaw/README.md`
- `examples/openclaw/config.yaml.example`

适合 Gateway、agent turn、tool dispatch 这类平台边界。

### 4. 可复制业务骨架

- `examples/business_skeleton/README.md`
- `examples/business_skeleton/config.yaml.example`
- `examples/business_skeleton/handlers/register.py`

适合你准备把示例复制进自己的业务包，按工程结构继续扩展。

## 配置怎么放

工程模式项目也统一放：

```text
your_project/config/config.yaml
```

工程模式通常至少会用到：

- `logicfp.handler_registrars`
- `logicfp.backend_type`
- `logicfp.instance_id`
- `logicfp.default_source`

示例自己的业务配置则放在各自 section：

- `example_services:`
- `openclaw:`
- `business:`

## 启动方式

工程模式推荐直接用 CLI：

```powershell
logicfp start --config config/config.yaml
```

如果你只是仓库内快速看效果，也可以继续用：

- `examples/run_demo.ps1`
- `examples/run_services_demo.ps1`

但复制到真实项目时，还是建议回到你自己的 `config/config.yaml`。
