# 用户模式返回结构说明

这页只解释用户模式在 `simple=False` 时的返回结构。

如果你想让 `logicfp` 在成功和失败时都返回一个统一 envelope，而不是失败时抛 `ProtectRuntimeError`，就用：

```python
@protect(simple=False)
```

## 1. 成功结构

成功时返回：

```python
{
    "ok": True,
    "result": ...,
    "context": {...},
}
```

字段说明：

- `ok`
  固定为 `True`
- `result`
  你的函数返回值
- `context`
  本次调用的上下文信息

`context` 当前固定包含这些键：

- `request_id`
- `trace_id`
- `user_id`
- `source`
- `timestamp`
- `headers`
- `metadata`

示例：

```python
{
    "ok": True,
    "result": {"value": 2},
    "context": {
        "request_id": "req-...",
        "trace_id": "trace-...",
        "user_id": None,
        "source": "decorator",
        "timestamp": "2026-03-30T12:00:00+00:00",
        "headers": {},
        "metadata": {},
    },
}
```

## 2. 失败结构

失败时返回：

```python
{
    "ok": False,
    "error": {
        "code": ...,
        "message": ...,
        "details": {...},
    },
    "context": {...},
}
```

字段说明：

- `ok`
  固定为 `False`
- `error.code`
  错误码，例如 `ERR_VALIDATION`、`ERR_LOGIC`
- `error.message`
  错误消息
- `error.details`
  详细错误信息，没有时是空字典
- `context`
  本次调用上下文，字段和成功结构一致

示例：

```python
{
    "ok": False,
    "error": {
        "code": "ERR_LOGIC",
        "message": "manual review required",
        "details": {},
    },
    "context": {
        "request_id": "req-...",
        "trace_id": "trace-...",
        "user_id": None,
        "source": "decorator",
        "timestamp": "2026-03-30T12:00:00+00:00",
        "headers": {},
        "metadata": {},
    },
}
```

## 3. 什么时候用 simple=False

适合这些情况：

- 你要把结果继续传给上游统一处理
- 你不想在每个调用点都写 `try/except`
- 你希望成功和失败都走统一字典结构

不适合这些情况：

- 你更喜欢 Python 异常风格
- 你希望失败立即中断，不继续向上游传递 envelope

## 4. 和 simple=True 的区别

- `simple=True`
  成功直接返回结果，失败抛 `ProtectRuntimeError`
- `simple=False`
  成功返回 success envelope，失败返回 error envelope

## 5. 当前版本建议

如果你要长期依赖返回结构，建议只依赖这些顶层 contract：

- 成功：`ok / result / context`
- 失败：`ok / error / context`
- 错误对象：`code / message / details`

不要依赖当前内部实现细节，只依赖这几个公开字段。
